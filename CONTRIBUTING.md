# Contributing to Off the Grid Hackathon Platform

Thank you for your interest in contributing!

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/hackathon-dotnet-SKYUX.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Follow the setup instructions in [READY_TO_RUN.md](READY_TO_RUN.md)

## Development Workflow

1. Make your changes
2. Test thoroughly (backend and frontend)
3. Commit with clear messages: `git commit -m "Add: feature description"`
4. Push to your fork: `git push origin feature/your-feature-name`
5. Create a Pull Request

## Code Standards

### Backend (.NET)
- Follow Microsoft C# coding conventions
- Use meaningful variable and method names
- Add XML documentation comments for public APIs
- Write unit tests for new features

### Frontend (Angular)
- Follow Angular style guide
- Use SKY UX components where possible
- Keep components focused and single-purpose
- Write unit tests for components and services

## Pull Request Guidelines

- Provide a clear description of changes
- Reference any related issues
- Ensure all tests pass
- Update documentation if needed
- Keep PRs focused on a single feature/fix

## Questions?

Open an issue for:
- Bug reports
- Feature requests
- Documentation improvements
- Questions about the codebase
